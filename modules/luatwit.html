Functions

new (args)
Creates a new api object with the supplied keys. An object created with only the consumer keys must call api:start_login and api:confirm_login to get the access token, otherwise it won't be able to make API calls.

Parameters:

  • args Table with the OAuth keys (consumer_key, consumer_secret, oauth_token, oauth_token_secret).

Returns:

    New instance of the api class.

See also:

    luatwit.objects.access_token
load_keys (...)
Helper to load OAuth keys from text files. Key files are loaded as Lua files in an empty environment and the values are extracted from their global namespace. It also accepts tables as arguments (useful when using require).

Parameters:

  • ... Filenames (Lua code) or tables with the keys to load.

Returns:

    Table with the keys found.

Returns:

_M.api:raw_call (decl, args, name, defaults)
Generic call to the Twitter API. This is the backend method that performs all the API calls.

Parameters:

  • decl API method declaration. This is taken from the resources table.
  • args Table with the method arguments.
  • name Method name. Used internally for building error messages.
  • defaults Default method arguments.

Returns:

  1. A table with the decoded JSON data from the response, or nil on error. If the option _raw is set, instead returns an unprocessed JSON string.
  2. HTTP headers. On error, instead it will be an string or luatwit.objects.error describing the error.
  3. If the option _raw is set, the type name from resources . This value is needed to use the api:parse_json with the returned string. If an API error ocurred, instead it will be the HTTP headers of the request.
_M.api:parse_json (str, tname)
Parses a JSON string and applies type metatables.

Parameters:

  • str JSON string.
  • tname Type name as defined in resources . If set, the function will set type metatables.

Returns:

    A table with the decoded JSON data.
_M.api:start_login ()
Begins the OAuth authorization. This method generates the URL that the user must visit to authorize the app and get the PIN needed for api:confirm_login.

Returns:

  1. Authorization URL.
  2. HTTP Authorization header.
_M.api:confirm_login (pin)
Finishes the OAuth authorization. This method receives the PIN number obtained in the api:start_login step and authorizes the client to make API calls.

Parameters:

  • pin PIN number obtained after the user succefully authorized the app.

Returns:

  1. An access_token object.
  2. HTTP Status line.
  3. HTTP result code.
  4. HTTP headers.
